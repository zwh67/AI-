# VoiceCanvas Lite 升级方案

## 1. 当前版本定位

当前版本已经能证明核心闭环：

```text
语音输入 -> 指令理解 -> LLM / 本地规则生成 actions -> Canvas 绘图 -> 语音继续编辑
```

它适合作为“纯语音 AI 绘图白板”的最终提交版本。优势是方向清晰、链路完整、具备演示价值；短板是图示质量、复杂编辑和 AI 稳定性仍有产品化空间。

## 2. 优先升级方向

### 2.1 图结构 Schema 升级

当前 actions 偏底层，直接描述 `draw_rect`、`draw_arrow`。下一阶段建议新增中间图结构：

```json
{
  "type": "flowchart",
  "nodes": [],
  "edges": [],
  "groups": [],
  "layout": "horizontal"
}
```

收益：

- LLM 先生成语义图结构，再由布局器转换为 Canvas actions。
- 更容易校验重复节点、孤立节点、错误连接。
- 支持分支、泳道、分组、条件判断等高级图示。

### 2.2 布局引擎升级

当前布局主要适合横向流程图。建议增加三类布局：

- 流程图：横向主链 + 分支回流。
- 架构图：分层布局，例如用户层、应用层、服务层、数据层。
- 关系图：中心辐射或左右实体关系。

收益：

- 生成图更专业。
- 扩容时能自动避让已有节点。
- 导出图片更像正式图表。

### 2.3 语音编辑能力升级

下一阶段重点支持：

- 插入：`在 A 和 B 之间插入 C`
- 移动：`把审批节点放到签合同前面`
- 分支：`从校验信息分出成功和失败两条线`
- 样式：`把失败节点改成红色`
- 合并：`把清洗和切菜合并成准备食材`

收益：

- 用户能真正完成多轮创作，而不是只生成一次。
- 语音交互更接近自然编辑。

### 2.4 AI 执行前确认

当前版本已经对部分模糊扩展加入基础确认机制：

```text
用户：复杂一点
系统：我建议添加“库存校验、物流跟踪、售后评价”，请说确认或取消
用户：确认
```

下一阶段建议把确认机制扩展为所有高风险 AI 编辑的通用预览：

```text
用户：复杂一点
系统：我准备添加“调味、摆盘、收拾餐具”三个节点，是否执行？
用户：确认
```

收益：

- 降低 LLM 随机扩展造成的破坏。
- 适合纯语音场景下的安全回滚。

### 2.5 语音识别稳定性升级

建议增加：

- 最近语音历史窗口。
- 置信度阈值。
- 低置信度二次确认。
- 同音词纠错，例如“扩容/扩充/扩展”归一化。

收益：

- 降低误触发。
- 提高无键鼠环境下的可用性。

## 3. 可提交版本边界

本次最终版本建议明确声明：

- 支持纯语音生成和编辑基础图示。
- 支持本地模板与 LLM 开放流程图。
- 支持撤回、清空、导出、帮助、停止监听等语音控制。
- 支持复杂指令拆解和 LLM 输出容错。
- 不承诺任意复杂图表的完美布局。
- 不承诺浏览器首次麦克风授权无需用户交互。

## 4. 后续迭代优先级

| 优先级 | 升级项 | 原因 |
| --- | --- | --- |
| P0 | 图结构 Schema | 解决 LLM 输出不可控和后续编辑困难 |
| P0 | 通用执行前预览 | 当前已有基础确认，后续扩展到所有高风险 AI 编辑 |
| P1 | 布局引擎 | 提升视觉质量和专业感 |
| P1 | 插入/移动/分支语音编辑 | 提升真实创作能力 |
| P2 | 多主题样式与导出模板 | 提升成品展示效果 |
| P2 | 长对话记忆 | 支持更复杂的连续创作 |

## 5. 竞争力判断

当前版本的竞争力来自“纯语音 + AI 图示规划 + Canvas 可编辑动作”的组合。它不是最终产品形态，但已经能作为课程项目、原型展示或作品集项目成立。

如果后续补齐图结构 Schema、布局引擎和执行前确认，产品竞争力会明显提升。
