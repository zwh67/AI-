# VoiceCanvas Lite 设计文档

## 1. 目标与约束

VoiceCanvas Lite 的目标是实现一个仅通过语音完成绘图创作的 AI Canvas 白板。用户不依赖鼠标和键盘，核心操作都通过自然语言完成：

```text
说出意图 -> 语音识别 -> 本地规则或 LLM 理解 -> 结构化 actions -> Canvas 绘制
```

本项目不让 LLM 直接生成图片，也不让 LLM 直接执行 Canvas API。LLM 只负责把自然语言规划成受控 JSON actions，前端再按动作白名单绘制。

## 2. 计划支持的指令能力

| 能力 | 计划指令示例 | 设计目标 |
| --- | --- | --- |
| 启动与状态 | 自动开始监听、停止监听、帮助 | 用户尽量不需要点击按钮 |
| 一句话生成 | 画一个员工入职流程图 | 从自然语言生成完整图示 |
| 模板生成 | 画登录流程图、画前后端架构图 | 低延迟稳定演示 |
| LLM 开放生成 | 画一个请假审批流程图 | 处理未内置的新主题 |
| 多轮扩展 | 复杂一点、扩容一下、添加短信验证码节点 | 基于当前图继续编辑 |
| 精确编辑 | 删除登录失败节点、把输入账号密码改成填写账号 | 语音选择与修改元素 |
| 控制操作 | 撤回、清空、导出图片 | 不依赖工具栏 |
| 教学示意 | 画平抛运动、做受力分析 | 保留教学模板能力 |
| 复杂指令拆解 | 画平抛，设置速度 5，显示轨迹，然后开始模拟 | 拆分并顺序执行多个动作 |

## 3. 最终实现情况

### 3.1 语音输入闭环

- 页面加载后会自动尝试启动语音监听。
- 浏览器不支持语音识别时，页面显示“语音不可用”。
- 用户可以说“帮助 / 怎么用 / 有哪些指令”查看可用语音指令。
- 语音识别的最终文本会自动写入当前绘图意图，后续可直接说“生成图示”。
- 为避免电脑外放声音被麦克风再次识别，监听期间默认不进行语音播报，只更新页面状态与日志。
- 对 2.5 秒内重复出现的同一句 final 识别结果做去重，减少重复绘图。

### 3.2 指令理解与容错

本地规则负责低延迟和高确定性的指令：

- `撤回 / 撤销 / 回退`
- `清空 / 重置画布`
- `导出 / 保存图片`
- `开始模拟 / 暂停 / 重播`
- `删除某节点`
- `把 A 改成 B`
- `简化这个图`
- 明确模板：登录流程、购物流程、前后端架构、AI 开发流程、平抛、斜面、圆周等

LLM 负责开放图示：

- 未内置流程图，例如员工入职、报销、请假、外卖、数据处理等。
- LLM 返回 `actions`、`nodes`、`steps`、`data.nodes`、Mermaid 等常见格式时，后端会尽量编译成 Canvas actions。
- 如果 LLM 返回非法 action 类型，后端会尝试从用户原句的“包括 A、B、C”中修复流程节点。
- LLM 输出始终经过动作白名单过滤。

### 3.3 多轮扩展

已支持两种扩展：

- 明确扩展：`添加短信验证码节点` 会直接追加具体节点。
- 模糊扩展：`复杂一点 / 扩容一下 / 丰富一下` 会把当前图的节点和箭头作为上下文发给 LLM。
- 对部分可由本地语义判断的模糊扩展，系统会先提出候选节点，例如“我建议添加：库存校验、物流跟踪、售后评价”，用户说“确认”后才执行，说“取消”则放弃。

为了避免乱画，扩展当前图时增加了约束：

- 禁止清空和重画整张图。
- 禁止重复已有节点。
- 禁止添加“一点 / 新步骤 / 节点”等占位词。
- 只允许追加真实业务节点，并自动接到当前流程末尾。
- 如果无法判断补什么，应请求用户说明具体环节。

### 3.4 响应延迟设计

- 本地模板和控制指令同步执行，响应通常在毫秒级。
- LLM 请求设置了前端超时，超时后会给出明确提示。
- 调用 LLM 时会更新状态为“正在扩展绘图意图”或显示 LLM 错误。
- 后端支持 `EDUSKETCH_DEBUG_LLM=1`，可打印 LLM 原始返回和解析结果，便于定位延迟或格式问题。

### 3.5 复杂指令拆解

本地解析器会拆分包含“然后、同时、再、接着、以及、和”等连接词的复合语句，并按顺序生成 actions。例如：

```text
画一个平抛运动，设置初速度 5，设置高度 3，显示速度方向，然后开始模拟
```

会拆成：

```text
draw_projectile
set_projectile_velocity
set_projectile_height
show_velocity
start_simulation
```

## 4. 未完成部分与原因

| 未完成内容 | 原因说明 |
| --- | --- |
| 完全无需任何首次交互的麦克风授权 | 浏览器安全策略要求用户授权麦克风；应用只能自动尝试启动，无法绕过权限提示 |
| 任意复杂版式的完美自动排版 | 当前布局以流程图横向/分行为主，复杂网状图需要更完整布局引擎 |
| 稳定的语音指代消解 | “把那个移到右边”“删掉第二个分支”需要更强的元素选择模型，目前支持文本匹配 |
| 长对话记忆 | 当前上下文主要来自当前画布和最近意图，未实现完整会话记忆 |
| 真实文生图 | 项目定位是结构化 Canvas 绘图，不是生成位图图片 |
| 高级几何编辑 | 支持基础图元和流程图，未实现拖拽式自由排版的语音替代方案 |

## 6. 总结

最终实现的主线是“语音到结构化绘图动作”的创作工具。它不是简单的命令画板，而是通过本地规则保证低延迟，通过 LLM 支持开放主题，通过动作白名单和扩展约束保证安全可控。
